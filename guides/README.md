# Guides

This folder is the GitHub-facing documentation hub for AeroTrace Turbofan. Start here if you want to understand the repository before diving into notebooks, apps, or reports.

## Start here

If you only open one document first, make it [`project-overview.md`](./project-overview.md). It explains the project flow, the major folders, and how the data, demo, web app, and twin pieces connect.

## Guide map

| Guide | Purpose | Open this when... |
| --- | --- | --- |
| [`project-overview.md`](./project-overview.md) | High-level architecture and repository flow | You want the big picture first |
| [`data-assets.md`](./data-assets.md) | Inventory of datasets, outputs, figures, and committed evaluation visuals | You want to know what is actually committed |
| [`validation-status.md`](./validation-status.md) | Evidence review, caveats, and current validation boundaries | You need an honest status check |
